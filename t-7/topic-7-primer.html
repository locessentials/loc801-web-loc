TOPIC 7

Translation Technology Options

Not all translation technology is created equal — and in many cases it's not right for web localization at all. Learn how Translation Environment Tools work, what their limitations are, and how to make informed choices about when and how to use them.

🎯 Learning Objectives

  • Understand how sentence-by-sentence segmentation works in CAT tools and why it creates specific challenges for web content
  • Predict the effects of segmentation on web localization quality before experiencing them hands-on
  • Evaluate Translation Environment Tools (TEnTs) using a structured scorecard designed for web localization contexts
  • Apply a CAT tool to localize a real webpage, including metadata, links, and content outside the main HTML file
  • Reflect critically on translation quality, SEO impact, and LLM discoverability after completing a hands-on localization project

📋 Action Items

Discussion
Segmentation & Web Localization

Before working with a CAT tool, predict how sentence-by-sentence segmentation will affect web localization quality — then compare your predictions to reality after the project.

Reference
Translation Technology Scorecard

Use a structured framework to evaluate Translation Environment Tools through the lens of web localization — covering file format support, collaboration, terminology management, and CMS integration.

Project
Python Patrol Localization Project

Use a CAT tool to localize a real conservation webpage into Miami Spanish, then reflect on translation quality, SEO impact, and what segment-by-segment translation actually produces.

⏱️ Estimated Time
Plan for approximately 6–8 hours to complete all activities in this topic. The Python Patrol project is the centerpiece — it combines your technical skills with critical reflection on translation quality, SEO, and the real-world limitations of CAT tool workflows.
