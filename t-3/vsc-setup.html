ðŸ“‹ VSC Setup Instructions

1

Install VS Code (If Needed)

This course first introduced to Visual Studio Code in Topic 2. If you didn't install it then, follow these steps to do so now:

  • Go to https://code.visualstudio.com
  • Click Download for your operating system
  • Install VS Code following the installer prompts
  • Launch VS Code
2

Enable Auto Save

Start off by enabling Auto Save to ensure you never lose your work. To do so:

  • Click on File in the menu bar
  • Select Auto Save
  • You'll see a checkmark appear next to Auto Save when it's enabled
3

Open a Project Folder

Important: You'll usually want to open an entire project folder, not just individual files. This gives you access to all your files in the sidebar and enables many helpful features. When you select "Open Folder", you'll only be able to select a folder, and individual files won't be displayed in the dialogue.

  • Click File > Open Folder
  • Navigate to your repository folder
  • Select the folder and click Open
  • You should now see your project files in the Explorer sidebar on the left
4

Enable Word Wrap

Word Wrap makes long lines of code readable by wrapping them to fit your screen instead of requiring horizontal scrolling:

  • Open any HTML file from the Explorer sidebar in your project
  • Click View in the menu bar
  • Select Word Wrap
  • Long lines will now wrap to fit your window
5

Install Live Server Extension

Live Server is essential for web developmentâ€”it opens a local web server where you can view your work locally on your machine and automatically refreshes your browser when you save changes:

  • Click the Extensions icon in the left sidebar
  • Search for "Live Server"
  • Find the extension by Ritwick Dey
  • Click Install
6

Preview Your Work with Live Server

Now you can preview any HTML file in your browser with live reload:

  • In the Explorer sidebar, right-click any HTML file
  • Select Open with Live Server
  • Your default browser will open showing the page
  • Any changes you make and save will automatically refresh the browser

Previewing your work this way allows you to see how your localization changes will appear on the actual webpage before deploying them live.
