Understanding HTML Code

üèóÔ∏è HTML Document Structure

Every HTML document follows a standard structure. Understanding this structure helps you identify where translatable content lives and where technical elements should remain unchanged.

The Two Main Sections
<head> - Contains metadata, the page title that shows in the browser tab, and links to stylesheets. This section is NOT visible to users but contains important SEO elements like tags and descriptions that often need localization. Search engine optimization (SEO) is covered in further detail in Topic 5.

<body> - Contains all the visible content that users see on the webpage. This is where most of the work of translation happens. 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Page Title</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>This is the visible content.</p>
</body>
</html>

Welcome to My Website


This is the visible content.

üîñ Opening and Closing Tags

Most HTML elements consist of an opening tag and a closing tag that wrap around content. The closing tag is identical to the opening tag except it has a forward slash (/) before the element name.

Tag Structure
Opening tag: <p> - Marks where the element begins

Content: The text or other elements that go inside

Closing tag: </p> - Marks where the element ends

Complete element: <p>This is my content</p>
üí° Critical for Localization
When translating HTML, you must keep both the opening and closing tags intact. Only translate the content between them. For example:

Original: <h1>Welcome to our site</h1>
Translated: <h1>La bienvenida a nuestro sitio</h1>

Notice that <h1> and </h1> remain unchanged‚Äîonly the text between them is translated.

üî§ Common HTML Elements

Headings

Headings structure content hierarchically. <h1> is the most important, <h6> is the least. Translate the heading text but maintain the heading level. 

<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Section Title</h3>
<h4>Subsection Title</h4>

Paragraphs

Paragraphs contain the main body text. Translate the content between <p> and </p> tags. 

<p>This is a paragraph of text.</p>
<p>This is another paragraph.</p>

This is a paragraph of text.

This is another paragraph.

Links

Links use the <a> tag. Translate the visible link text but be careful with the href attribute‚Äîsometimes URLs need localization, sometimes they don't. 

<a href="https://example.com">Visit our website</a>
<a href="/about.html">Learn more about us</a>
Visit our website
Learn more about us

Lists

Unordered lists (<ul>) show bullet points. Ordered lists (<ol>) show numbers. Each list item uses <li> tags. 

<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ul>

<ol>
    <li>Step one</li>
    <li>Step two</li>
    <li>Step three</li>
</ol>
  • First item
  • Second item
  • Third item
  1. Step one
  2. Step two
  3. Step three

Text Emphasis

<strong> makes text bold for importance. <em> makes text italic for emphasis. Preserve these formatting tags in translation. 

<p>This is <strong>very important</strong> text.</p>
<p>This is <em>emphasized</em> text.</p>

This is emphasized text.

Containers

<div> is a block-level container for grouping content. <span> is an inline container for styling parts of text. These don't change appearance by themselves but are often styled with CSS. 

<div>
    <p>This paragraph is inside a div.</p>
</div>

<p>This text has a <span>highlighted part</span>.</p>

üí° Key Takeaways for Localization

  • HTML tags themselves should never be translated‚Äîonly the content between opening and closing tags
  • The <head> section contains metadata like page titles and descriptions that are crucial for SEO and should be localized
  • The <body> section contains all visible content that users see and needs translation
  • Formatting tags like <strong> and <em> should generally be preserved in translation to maintain the intended emphasis
  • Link text (between <a> tags) should be translated, but href URLs require careful consideration
  • List structure (<ul>, <ol>, <li>) should be maintained even as you translate the list content

Find a simple webpage in English and view its source code (right-click ‚Üí "View Page Source" in most browsers). Identify:

  • Which HTML tags shouldn't be changed during translation
  • Any attributes (like href or alt text) that might need localization
