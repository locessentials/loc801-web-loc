TOPIC 10

Portfolio Project

Build and publish a bilingual portfolio website that showcases the projects you've completed throughout this course, a practical resource you can use when looking for clients, applying for new roles, or demonstrating your expertise at work.

🎯 Project Overview

You've spent this course learning how the web works, how to localize HTML, how to optimize content for search engines and large language models, how to work with translation tools, and how to think critically about language and audience. Now it's time to bring all of that together in a single, public-facing deliverable: your portfolio.

This portfolio project shows how you can represent yourself professionally, along with your skills, the work you've done, and how to reach you. When built with attention to localization, it also demonstrates that you understand how to make web content accessible and relevant across languages and cultures.

For this project, you'll fork a bilingual portfolio template, personalize it with your own content and projects, and publish it using GitHub Pages. The result will be a live website that you can link to from your résumé, LinkedIn profile, or job applications.

What You'll Be Publishing
A bilingual (English / Spanish) portfolio website hosted on GitHub Pages, featuring your projects from this course, your skills, your contact information, and a personal introduction, all customized with your own color palette, images, and content.

🍴 Step 1: Fork the Template Repository

The starting point for your portfolio is a pre-built template that you can customize to your experience and stylistic preferences.

1

Navigate to the Repository

Go to the locessentials/portfolio-template repository on GitHub.

2

Click Fork

In the upper right corner of the repository page, click the Fork button. On the next screen, change the name of the repo to just portfolio, keep the default settings and click Create fork. The repository will appear under your own GitHub account.

3

Open the Files in Your Code Editor

Clone your forked repository to your computer and open it in VS Code (or your editor of choice). You'll work primarily in index.html, inicio.html, and styles.css.

✏️ Step 2: Personalize Your Content

Open both index.html (English) and inicio.html (Spanish) and replace all the placeholder content with your own. Keep both versions in sync as you go.

Navigation Bar

Replace Full name | Title / Nombre completo | Título with your actual name and a short professional title. For example: Ana Morales | Web Localizer.


Portrait

Replace Portrait_Placeholder.png with a photo of yourself. A square or nearly square crop works best given the circular frame used in the template. If you'd rather not use a personal photo, a professional avatar or illustrated portrait is perfectly fine.

Image File Names
Keep your image file names simple, lowercase, and without spaces. For example: my-name-portrait.png or portfolio-portrait.png. This avoids broken links and encoding issues, especially on case-sensitive servers like GitHub Pages.

About Me

Replace the lorem ipsum placeholder with a short personal introduction. Who are you? What drew you to web localization? What languages do you work with? This doesn't need to be long. Three to five sentences is enough for a portfolio page.


Project Cards

Each project card has a title, a description, an image, and a link. Replace the placeholders with the projects you've completed in this course. For each one:

  • Add the project name
  • Write a short description of what you built and what skills it involved
  • Add a picture to go along with the project (replace Project_Placeholder.png)
  • Update the alt text on the image so that it's described accurately
  • Update the link to point to your live GitHub Pages site for that project

Skills Section

Update the skill categories and items to reflect what you actually know. The template includes categories for natural languages, translation, programming languages, and technology, but you can rename, add, or remove categories as needed.


Contact Section

Replace the placeholder contact details with your real email address, GitHub username, and LinkedIn URL (if you have one). You can leave out any fields you'd prefer not to share publicly.


Footer

Update the footer to include your name and the name of your institution (if applicable).

🎨 Step 3: Customize Your Color Palette

The template's default colors are functional placeholders, but choosing your own palette is one of the most impactful ways to make the portfolio feel like yours. All color values live in a single place: the :root block at the top of styles.css.

You'll find separate variables for light mode and dark mode. You only need to change the hex values — the rest of the stylesheet updates automatically.

Color Selection Tools

If you're not sure where to start, these free tools can help you find a palette that works:

  • Adobe Color - explore color harmonies visually
  • Canva Color Tools - get ideas for color combinations
  • Coolors - generate and lock palette combinations quickly
  • WebAIM Contrast Checker - verify that your text is readable against its background

As a general rule: give each color a role (background, surface, accent, text), make sure text and background have enough contrast (at least 4.5:1 for body text), and think about how your light and dark modes relate to each other. They don't need to be simple inversions.

🚀 Step 4: Publish with GitHub Pages

Once your portfolio looks the way you want, it's time to publish it.

1

Commit and Push Your Changes

Make sure all your changes are saved, committed, and pushed to your forked repository on GitHub.

2

Open Repository Settings

Go to your forked repository on GitHub and click the Settings tab near the top of the page.

3

Enable GitHub Pages

In the left sidebar, click Pages. Under "Source," select Deploy from a branch. Set the branch to main and the folder to / (root). Click Save.

4

Wait for Deployment

GitHub Pages takes a minute or two to build and deploy. Once it's ready, your site will be live at:

https://yourusername.github.io/portfolio/

🔍 Step 5: Review Your Localization

Before submitting, take a careful look at both language versions of your portfolio. This is for a course on web localization, so the quality of your work matters.

Check that all visible text has been translated (including alt text on images), that the lang attribute is correctly set to es-MX on inicio.html, that the language toggle links work in both directions, and that the text reads naturally in both lanuages, not like a word-for-word translation.

Localization Quality Checklist

  • All visible text has been written, including navigation, section headings, button labels, and footer
  • Language toggle links working correctly in both directions
  • Alt text on all images translated
  • The CSS looks polished and professional
  • No lorem ipsum text remaining

💬 Reflection

Along with the link to your published portfolio, submit a short written reflection addressing the following questions:

1. Which projects did you choose to feature, and why?

You may not have completed every project in the course, or you may have completed some better than others. Tell us which ones you decided to highlight and what guided that decision.

2. What localization decisions did you make when writing each of the versions?

How did you approach writing your personal introduction and project descriptions? Were there any terms or phrases that were particularly tricky to handle? Did you make any intentional decisions about register, inclusive language, or regional conventions?

3. What design or color choices did you make, and how did you approach light vs. dark mode?

Describe your process for choosing your color palette. Did you start from a brand, a feeling, or a set of tools? How did you ensure sufficient contrast?

4. What would you do differently if you had more time?

This is a chance to reflect honestly on the work. What feels incomplete or rough around the edges? What would a polished, fully production-ready version of this portfolio include?

5. Anything else you'd like to add?

Any challenges you encountered, things you're proud of, or observations worth sharing.

🌱 Want to Do Something Different?

The portfolio template is the default capstone project for this course, but it isn't the only way to showcase your web localization skills. If you have a different idea that excites you more, you're welcome to propose it.

Some examples of alternative projects that would demonstrate comparable skills:

  • A bilingual product page for a real or fictional product, designed with a specific regional audience in mind
  • Two blog posts on a similar topic — one written in U.S. English, one in Mexican Spanish — with a write-up comparing the keyword research and SEO strategy for each audience
  • A localized landing page for a non-profit or community organization, with documentation of your translation and localization decisions
  • Any other project that combines HTML, bilingual content, and thoughtful localization decisions
Check with Your Instructor First
If you're completing this course as part of a class, get your instructor's approval before embarking on a self-designed capstone project. They'll want to make sure the scope and deliverables are comparable to the standard portfolio assignment.
