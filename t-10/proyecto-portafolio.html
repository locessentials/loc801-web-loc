TEMA 10

Proyecto Portafolio

Crea y publica un portafolio web bilingüe que muestre los proyectos que completaste a lo largo del curso, un recurso práctico que puedes usar al buscar clientes, postularte a nuevos puestos o demostrar tu experiencia en el trabajo.

🎯 Descripción del Proyecto

A lo largo de este curso aprendiste cómo funciona la web, cómo localizar HTML, cómo optimizar contenido para motores de búsqueda y modelos de lenguaje a gran escala, cómo trabajar con herramientas de traducción y cómo pensar de manera crítica sobre el lenguaje y el público destinatario. Ahora es momento de reunir todo eso en un producto final público: tu portafolio.

Este proyecto de portafolio muestra cómo te puedes presentar profesionalmente: tus habilidades, el trabajo que has realizado y cómo contactarte. Cuando se construye con atención a la localización, también demuestra que entiendes cómo hacer que el contenido web sea accesible y relevante en distintos idiomas y culturas.

Para este proyecto, bifurcarás una plantilla de portafolio bilingüe, la personalizarás con tu propio contenido y proyectos, y la publicarás usando GitHub Pages. El resultado será un sitio web en vivo al que podrás enlazar desde tu currículum, tu perfil de LinkedIn o tus solicitudes de empleo.

Lo que Vas a Publicar
Un portafolio web bilingüe (inglés / español) alojado en GitHub Pages, con tus proyectos del curso, tus habilidades, tu información de contacto y una presentación personal, todo personalizado con tu propia paleta de colores, imágenes y contenido.

🍴 Paso 1: Bifurca el Repositorio de la Plantilla

El punto de partida para tu portafolio es una plantilla prediseñada que puedes personalizar para tu experiencia y preferencias estilísticas.

1

Navega al Repositorio

Ve al repositorio locessentials/portfolio-template en GitHub.

2

Haz Clic en Fork

En la esquina superior derecha de la página del repositorio, haz clic en el botón Fork. En la siguiente pantalla, cambia el nombre del repo a solo la palabra portfolio, deja los valores predeterminados y haz clic en Create fork. El repositorio aparecerá bajo tu propia cuenta de GitHub.

3

Abre los Archivos en tu Editor de Código

Clona el repositorio bifurcado en tu computadora y ábrelo en VS Code (o el editor de tu preferencia). Trabajarás principalmente en index.html, inicio.html y styles.css.

✏️ Paso 2: Personaliza el Contenido

Abre tanto index.html (inglés) como inicio.html (español) y reemplaza todo el contenido de relleno con el tuyo. Mantén ambas versiones sincronizadas a medida que avanzas.

Barra de Navegación

Reemplaza Nombre completo | Título / Full name | Title con tu nombre real y un título profesional breve. Por ejemplo: Ana Morales | Localizadora Web.


Retrato

Reemplaza Portrait_Placeholder.png con una foto tuya. Un recorte cuadrado o casi cuadrado funciona mejor dado el marco circular que usa la plantilla. Si prefieres no usar una foto personal, un avatar profesional o un retrato ilustrado está perfectamente bien.

Nombres de Archivos de Imagen
Mantén los nombres de tus archivos de imagen simples, en minúsculas y sin espacios — por ejemplo, mi-nombre-retrato.jpg o retrato-portafolio.png. Esto evita enlaces rotos y problemas de codificación, especialmente en servidores que distinguen entre mayúsculas y minúsculas, como GitHub Pages.

Sobre Mí

Reemplaza el texto de relleno lorem ipsum con una breve presentación personal. ¿Quién eres? ¿Qué te llevó a la localización web? ¿Con qué idiomas trabajas? No tiene que ser larga — tres a cinco oraciones son suficientes para una página de portafolio.


Tarjetas de Proyectos

Cada tarjeta de proyecto tiene un título, una descripción, una captura de pantalla y un enlace. Reemplaza los datos de relleno con los proyectos que completaste en este curso. Para cada uno:

  • Agrega el nombre del proyecto
  • Escribe una descripción breve de lo que construiste y qué habilidades involucró
  • Agrega una imagen para acompañar el proyecto (reemplaza Project_Placeholder.png)
  • Actualiza el texto alt de la imagen para que la describa con precisión
  • Actualiza el enlace para que apunte a tu sitio de GitHub Pages para ese proyecto

Sección de Habilidades

Actualiza las categorías y elementos de habilidades para que reflejen lo que realmente sabes. La plantilla incluye categorías para idiomas naturales, traducción, lenguajes de programación y tecnología, pero puedes renombrar, agregar o eliminar categorías según sea necesario.


Sección de Contacto

Reemplaza los datos de contacto de relleno con tu correo electrónico real, tu nombre de usuario de GitHub y tu URL de LinkedIn (si tienes). Puedes omitir los campos que prefieras no compartir públicamente.


Pie de Página

Actualiza el pie de página para incluir tu nombre y el nombre de tu institución (si aplica).

🎨 Paso 3: Personaliza tu Paleta de Colores

Los colores predeterminados de la plantilla son marcadores de posición funcionales, pero elegir tu propia paleta es una de las formas más efectivas de hacer que el portafolio se sienta tuyo. Todos los valores de color se encuentran en un solo lugar: el bloque :root al inicio de styles.css.

Encontrarás variables separadas para el modo claro y el modo oscuro. Solo necesitas cambiar los valores hexadecimales — el resto de la hoja de estilos se actualiza automáticamente.

Herramientas para Elegir Colores

Si no sabes por dónde empezar, estas herramientas gratuitas pueden ayudarte a encontrar una paleta que funcione:

  • Adobe Color - explora armonías de color visualmente
  • Canva Color Tools - encuentra inspiración para combinaciones de colores
  • Coolors - genera y combina paletas rápidamente
  • WebAIM Contrast Checker - verifica que el texto sea legible sobre el fondo elegido

Como regla general: asigna a cada color un rol (fondo, superficie, acento, texto), asegúrate de que el texto y el fondo tengan suficiente contraste (al menos 4.5:1 para el cuerpo del texto) y piensa en cómo se relacionan el modo claro y el modo oscuro entre sí. No tienen que ser simples inversiones.

🚀 Paso 4: Publica con GitHub Pages

Una vez que tu portafolio luzca como quieres, es hora de publicarlo.

1

Confirma y Sube tus Cambios

Asegúrate de que todos tus cambios estén guardados, confirmados (committed) y subidos (pushed) a tu repositorio bifurcado en GitHub.

2

Abre la Configuración del Repositorio

Ve a tu repositorio bifurcado en GitHub y haz clic en la pestaña Settings (Configuración) cerca de la parte superior de la página.

3

Activa GitHub Pages

En la barra lateral izquierda, haz clic en Pages. En "Source" (Fuente), selecciona Deploy from a branch. Configura la rama en main y la carpeta en / (root). Haz clic en Save.

4

Espera el Despliegue

GitHub Pages tarda uno o dos minutos en compilar y desplegar el sitio. Una vez listo, tu portafolio estará disponible en:

https://tunombredeusuario.github.io/portfolio/

🔍 Paso 5: Revisa tu Localización

Antes de entregar, revisa con cuidado ambas versiones lingüísticas de tu portafolio. Este es un curso de localización web, así que la calidad de tu trabajo importa.

Verifica que todo el texto visible esté traducido (incluyendo el texto alternativo de las imágenes), que el atributo lang esté correctamente configurado como es-MX en inicio.html, que los enlaces del selector de idioma funcionen en ambas direcciones y que el texto se lea con naturalidad en ambas idiomas, no como una traducción palabra por palabra.

Lista de Verificación de Calidad de Localización

  • Todo el texto visible escrito, incluyendo navegación, encabezados de sección, etiquetas de botones y pie de página
  • Selector de idioma funcionando correctamente en ambas direcciones
  • Texto alternativo de todas las imágenes traducido
  • El CSS se ve pulido y profesional
  • No queda texto de "lorem ipsum"

💬 Reflexión

Junto con el enlace a tu portafolio publicado, entrega una breve reflexión escrita que responda las siguientes preguntas:

1. ¿Qué proyectos elegiste destacar y por qué?

Es posible que no hayas completado todos los proyectos del curso, o que algunos te hayan salido mejor que otros. Cuéntanos cuáles decidiste incluir y qué guió esa decisión.

2. ¿Qué decisiones de localización tomaste al escribir cada una de las versiones?

¿Cómo abordaste tu presentación personal y las descripciones de proyectos? ¿Hubo términos o expresiones que resultaron especialmente difíciles de manejar? ¿Tomaste decisiones intencionales sobre el registro, el lenguaje inclusivo o las convenciones regionales?

3. ¿Qué decisiones de diseño o paleta de colores tomaste, y cómo abordaste el modo claro vs. modo oscuro?

Describe tu proceso para elegir la paleta de colores. ¿Partiste de una marca, un sentimiento o un conjunto de herramientas? ¿Cómo verificaste que el contraste fuera suficiente?

4. ¿Qué harías de manera diferente si tuvieras más tiempo?

Esta es una oportunidad para reflexionar con honestidad sobre el trabajo. ¿Qué se siente incompleto o poco pulido? ¿Qué incluiría una versión totalmente terminada y lista para producción de este portafolio?

5. ¿Quieres agregar algo más?

Cualquier dificultad que hayas encontrado, algo de lo que estés orgullosa u orgulloso, o cualquier observación que valga la pena compartir.

🌱 ¿Quieres Hacer Algo Diferente?

La plantilla de portafolio es el proyecto final predeterminado para este curso, pero no es la única manera de mostrar tus habilidades en localización web. Si tienes una idea diferente que te entusiasma más, puedes proponerla.

Algunos ejemplos de proyectos alternativos que demostrarían habilidades comparables:

  • Una página de producto bilingüe para un producto real o ficticio, diseñada pensando en un público regional específico
  • Dos entradas de blog sobre un tema similar — una escrita en inglés estadounidense y otra en español mexicano — con un análisis comparando la investigación de palabras clave y la estrategia de SEO para cada público
  • Una página de destino localizada para una organización sin fines de lucro o comunitaria, con documentación de tus decisiones de traducción y localización
  • Cualquier otro proyecto que combine HTML, contenido bilingüe y decisiones de localización bien fundamentadas
